1 - Como escrever livros com o Tubaina?

1.1 - Criação de capítulos

Todos os novos capítulos, devem ser identificados com a Tag [chapter]. Um arquivo só pode ter um chapter. No caso desse capítulo, utilizamos o seguinte:


        [chapter Como escrever livros com o Tubaina?]
    

1.2 - Criação de seções

As seções podem ser criadas a partir do uso da Tag section, contendo o nome da seção a frente. No caso dessa seção, fica:


        [section Criação de seções]
            Em seguida vai o conteúdo da seção...
    

1.3 - Quebras de linha e de página

As quebras de linha devem ser especificadas no arquivo .afc com duas quebras (ou seja, dois enter). Uma só quebra é desconsiderada, pois admite-se que é apenas para organização do arquivo fonte. Portanto, sempre que quiser um novo parágrafo (como o abaixo), utilize duas quebras de linha.

As quebras de página são feitas automaticamente no build do livro.


1.4 - Títulos

Para criar título para um assunto dentro de uma seção, pode-se utilizar a Tag title contendo o título que será utilizado.

Conclusão e como usar

Com isso, basta utilizar em algum lugar do seu arquivo .afc o seguinte conteúdo:


        [title Conclusão e como usar]
    

1.5 - Definição de boxes

Quando quiser detalhar algo que está no texto, sem perder a linearidade do mesmo, recomendamos o uso de um Box, onde é possível adicionar algum conteúdo, como a seguir:

Linearidade do texto

Ao apresentarmos uma ideia, devemos manter uma sequência lógica em nosso texto...

Criar um box é extremamente simples, bastando que definamos uma Tag box contendo um título e o conteúdo esteja dentor da Tag. No caso do anterior, tivemos o seguinte:


        [box Linearidade do texto]
            Ao apresentarmos uma ideia, devemos manter uma sequência lógica em noss 
            texto...
        [/box]
    

1.6 - Índice remissivo

O índice remissivo é criado a partir da definição do autor dos pontos de índice. Para isso, em qualquer lugar do texto, basta utilizar a Tag %%

[index]%% contendo o nome do índice. Automaticamente o número da página é descoberto e colocado no índice.

No caso dessa seção, foi definido um índice que pode ser visto no final do livro. Para isso, utilizamos o seguinte:


        [index Índice]
    

1.7 - Como colocar imagens

Para adicionar imagens, basta utilizar a Tag img indicando qual o nome do arquivo que deseja importar. Lembre-se de que o caminho do arquivo da imagem, deverá ser relativo ao diretório onde se encontra o arquivo .afc.

Também é possível dizer a proporção que deve ser utilizada para a imagem através de uma porcentagem.

Um exemplo de uso é:


        [img caminho/do/arquivo/de/imagens w=100%]
    

É recomendado também adicionar legenda nas fotos. Para isso, basta adicionar um texto entre aspas:


        [img caminho/do/arquivo/de/imagens w=90% "imagem com legenda"]
    

1.8 - Criação de To-Dos

Você pode criar to-dos, que são lembretes de algo que você precisa fazer depois. Os textos marcados como To-Dos não saem na impressão, é só são visualizadas no arquivo fonte .afc.

A imagem permite visualizar um candlestick.

Para adicionar um To-Do, utilize a Tag TODO:


        [TODO Algo que eu tenho que me lembrar de re-ler depois]
    

Esses To-Dos aparecem no build do livro, junto do pdf da versão buildada.


2 - Mais detalhes da sintaxe do tubaina

"Dá-se importância aos antepassados quando já não temos nenhum."
--François Chateaubriand

2.1 - Adicione citações

É possível adicionar citações adicionando a Tag quote com o texto em seguida. No exemplo desse capítulo, foi adicionado o seguinte conteúdo:


        [quote "Dá-se importância aos antepassados quando já não temos nenhum." 
            -- François Chateaubriand]
    

2.2 - Trabalhando com itálico, negrito e código inline

Quando quiser que um texto apareça em itálico, basta adicioná-lo entre : : conteúdo aqui : : (dois dois pontos), dessa forma, o texto ficará em itálico.

Para fazer com que um treco do texto fique em negrito, coloque-o entre * * conteúdo aqui * * (dois asteriscos), com isso, o texto ficará em negrito.

Quando precisamos utilizar no meio de um texto algum conteúdo de código, citar algum arquivo, nome de classe ou algo do tipo, sem gerar uma quebra de linha para isso, ou seja, inline, podemos utilizar % % codigo aqui % % (dois %). Um exemplo é: <html></html>. Esse texto terá fonte monoespaçada.


2.3 - Organize informações com listas

Se deseja enumerar itens, pode-se utilizar uma lista numerada, através da Tag list com o atributo number. Cada item da lista deve ser identificado com um * no início:

  1. Primeiro item;
  2. Segundo item;
  3. Terceiro item;

Também é possível utilizar letras para ordenas (útil para criar sublistas):

  1. Item A;
  2. Item B;
  3. Item C;

Caso não deseja uma lista ordenada, ou seja, sua lista terá apenas os bullets, omita o tipo da lista:

  • Bullet 1
  • Bullet 2
  • Bullet 3

Nos casos anteriores, tivemos o seguinte código:


        [list number]
            * Primeiro item;
            * Segundo item;        
            * Terceiro item;
        [/list]
        
        [list letter]
            * Item A;
            * Item B;
            * Item C;
        [/list]
        
        [list]
            * Bullet 1
            * Bullet 2
            * Bullet 3
        [/list]
    

2.4 - Escrevendo códigos

É possível escrever código em diversas linguagens e fazer com que eles tenham syntax highlight adequado. Para isso, basta usar a Tag code especificando uma linguagem.

Exemplo:


        [code java]
            double variavel;
            System.out.println("oi");
        [/code]
    

E isso gerará o seguinte conteúdo:


        double variavel;
        System.out.println("oi");
    

Para a identação do código será utilizada a mesma que for colocada dentro da Tag code.

Caso seja necessário identificar o número da linha, é possível adicioná-las colocando uma # na Tag code:


        [code java #]
            System.out.println("linha 1");
            System.out.println("linha 2");
            System.out.println("linha 3");
        [/code]
    

Que gera:


            System.out.println("linha 1");
            System.out.println("linha 2");
            System.out.println("linha 3");
        

Também é possível destacar linhas do código, utilizando o atributo h na Tag code:


        [code java # h=2]
            System.out.println("linha 1");
            System.out.println("linha 2");
            System.out.println("linha 3");
        [/code]
    

Que gerará:


        System.out.println("linha 1");
        System.out.println("linha 2");
        System.out.println("linha 3");
    

Para uma lista das linguagens disponíveis, veja: http://pygments.org/docs/lexers/

Caso precise de uma nova linguagem que não esteja disponível ainda, contate os editores.

Se necessário, é possível utilizar um trecho de código sem formatação alguma. Para isso, basta omitir a linguagem.

Você pode utilizar a Tag code dentro de qualquer lugar, como por exemplo, um Box, ou uma lista.

Cuidado pois existe um limite de caracteres em uma linha para o código do livro. No caso dos livros, esse limite e de 73 caracteres, para se encaixar bem na página. O autor deve quebrar o código onde achar mais conveniente. Caso o limite de caracteres seja estourado, o build falhará.


3 - Usando referências bibliográficas e para o próprio livro

"Oh! Como era difícil trabalhar com editoras tradicionais."
--Paulo Silveira

3.1 - Adicione referências

Fica fácil criar uma bibliografia sólida e consistente. Aliás, é importante sempre citar outras fontes que sustentem suas afirmações, em especial opiniões. Se for algo muito polêmico, compensa também citar fontes que pensam diferentemente e deixar o olhar crítico do leitor agir. Primeiramente, dentro do arquivo book.bib, você deve criar o livro ou site que quer se referenciar. Olhe esses dois exemplos:


        @article{getters-setters-silveira,
          author =      {Paulo Silveira},
          title =      {Como não aprender orientação a objetos: getters
             e setters},
          year =      {2006},
          howpublished = 
            {\url{http://www.iturls.com/english/expert/craig/craig1.pdf}}
        }

        @book{lean-startup-ries,
          author =      {Eric Ries},
          title =      {Lean Startup},
          year =      {2011},
          publisher = {Editora Convencional},
          howpublished = {\url{http://www.theleanstartup.com/book}}
        }
    

Para se referenciar, basta fazer:


        ... esses recursos são realmente fundamentais e transformaram a vida 
        de muitas empresas [ref lean-startup-ries].
    

Isso ficará assim: "esses recursos são realmente fundamentais e transformaram a vida de muitas empresas [ref lean-startup-ries]".

Verifique no fim desse PDF como a bibliografia aparecerá! Isso pode ser alterado, claro, para se utilizar iniciais dos autores em vez de números, aparecer primeiro ano e depois editora, etc.

Quando for uma primeira citação a um livro importante, pode ser interessante escrever o nome do livro: "O termo lean startup foi cunhado por Eric Ries em seu livro seminal Lean Startup [ref lean-startup-ries].". A não ser em casos específicos, não há necessidade de ficar escrevendo o nome do livro/blog/site toda hora! Evite também utilizar a bibliografia como conteúdo na frase. Isto é, evite: "Os getters e setters são ruins, como visto em [ref getters-setters-silveira]", prefira "Os getters e setters já são considerados má prática por muitos desenvolvedores [ref getters-setters-silveira]::".

Você pode se referenciar a dois itens dentro de uma mesma tag ref, separando por virgula (ref getters-setters-silveira, lean-startup-ries ), gerando [ref getters-setters-paulo-silveira, lean-startup-ries].


3.2 - Referenciando uma seção ou capítulo

Você gostou desse nosso capítulo de número 3? Já aprendeu a escrever referência bibliográfica na seção 3.1?

Mas e se adicionarmos um capítulo ou seção nova, teremos o trabalho de procurar essa citações, tipo "como vimos no capítulo X"? Seus problemas acabaram! Basta, dentro de uma seção ou capítulo, criar um label. Por exemplo:


    [ label adicione-referencias ]

E agora, para se referenciar, basta fazer:


    Você gostou do capítulo [ref-label capitulo-referencias]? Já aprendeu 
    a escrever referência bibliográfica na seção 
    [ref-label adicione-referencias]?

Ele mesmo saberá se a referência é a um capítulo ou seção, dependendo de onde você criou o label (se for antes de qualquer seção, será usada a numeração de capítulo).

Há também como se referenciar a uma página, mas não utilizaremos esse recurso pois formatos como epub e mobi podem não trabalhar bem com a ideia de uma página (kindle, por exemplo).


3.3 - Referenciando uma imagem

Para referenciar uma imagem você deve definir um label para a imagem na tag img. Depois, para referenciar a imagem no texto, use a tag ref-label, como no seguinte exemplo:


    [img resources/candle.jpeg w=50 "imagem de um candlestick" 
        label=candlestick]
    A imagem [ref-label candlestick] exibe um candlestick.
  

Que resulta na seguinte imagem, seguida pelo texto:

A imagem 3.3.1 exibe um candlestick.

A imagem 3.3.2 exibe um candlestick.


3.4 - Referenciando trechos de código

A imagem 3.4.1 exibe um candlestick.

Agora você também pode referenciar trechos de código do livro, para isso, você também deve definir um label na tag code. Para referenciá-lo no texto, use a tag ref-code, por exemplo:


    [code label=world]
      System.out.println("Hello world!");
    [/code]
    O execução do código na listagem [ref-code world] resulta em...
  

Que resulta no seguinte texto:


    System.out.println("Hello world!");
  

O execução do código na listagem 3.4.2 resulta em...
